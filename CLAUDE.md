# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Stack

- SvelteKit 2 + Svelte 5 (runes mode enforced project-wide) + Vite 8 + TypeScript 6
- Tailwind CSS 4 via `@tailwindcss/vite`
- `@sveltejs/adapter-cloudflare` → `.svelte-kit/cloudflare`, deployed to Cloudflare Pages
- Bun as package manager and runtime; lockfile is `bun.lock`
- Vitest 4 with browser-playwright + Playwright for E2E

## Commands

```sh
bun install --frozen-lockfile   # install
bun dev                         # dev server
bun run build                   # build → .svelte-kit/cloudflare
bun run preview                 # serve production build on :4173
bun run check                   # svelte-kit sync + svelte-check
bun run gen                     # regenerate worker-configuration.d.ts after wrangler.jsonc changes
```

A `justfile` at the repo root provides `just fmt`, `just lint`, `just build`, `just test`; `just` alone runs all four. Requires just ≥ 1.37.

## Lint & format

CI order is the source of truth (`.github/workflows/ci.yml`):

1. `bunx oxfmt --check .` — TypeScript/JS formatting
2. `bunx dprint check` — JSON/Markdown formatting (`dprint.json` configures plugins)
3. `bunx --bun oxlint --deny-warnings .` — JS/TS linting (`--bun` flag is required)
4. `bunx svelte-kit sync && bunx svelte-check --tsconfig ./tsconfig.json` — type check
5. `bun run lint:css` — Stylelint on `src/**/*.css`
6. `bun run lint:svelte` — ESLint on `.svelte` files (flat config in `eslint.config.js`)
7. `bunx knip` — unused exports/files/dependencies
8. `bun run lint:md` — markdownlint-cli2
9. `bunx --bun cspell "**/*.{md,svelte,ts,js}" --no-progress` — spell check

Local git hooks (lefthook) auto-fix staged files: `bunx lefthook install` to enable.

## Tests

- Unit: `bun run test:unit` (watch) / `bun run test:unit:coverage` (run with v8 coverage)
  - Two Vitest projects:
    - `client` — browser-playwright (chromium headless), `src/**/*.svelte.{test,spec}.{js,ts}`
    - `server` — Node environment, `src/**/*.{test,spec}.{js,ts}` (excludes `*.svelte.*`)
- E2E: `bun run test:e2e` — Playwright, matches `**/*.e2e.{ts,js}`; builds and serves on :4173

## Architecture

- `src/routes/` — file-system routes; `+page.svelte`, `+layout.svelte`, etc.
- `src/lib/` — shared library code, imported as `$lib/...`; never import `src/lib/server/` from client code
- `src/app.html` — page shell template
- `src/app.css` — global styles entry point (`@import "tailwindcss"`)
- `static/` — static assets copied as-is to build output
- `worker-configuration.d.ts` — Cloudflare env types generated by `bun run gen`; commit after changes to `wrangler.jsonc` bindings

## CI / deploy

Three-stage pipeline: **audit** (dependency review + Trivy) → **lint** → **build & test** → **deploy**.

Deploy to Cloudflare Pages requires: `CLOUDFLARE_API_TOKEN` (secret), `CLOUDFLARE_ACCOUNT_ID` (secret), `CF_PAGES_PROJECT` (variable). Deploy is a no-op until these are set; the rest of CI stays green.
